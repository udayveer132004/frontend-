# Homepage Design Brief (After Resume Upload)

## Context
This is the homepage shown immediately after the user uploads a resume.
Resume parsing happens in the background, so there is no "Resume Parsing" service box on this screen.

The homepage should mainly present 5 service boxes that represent the core features the user can use next.

## What the Designer Should Understand
This product is an AI Job Assistant for job seekers. After resume upload, the app already has user profile context from the resume and guides the user to 5 actions:
1. ATS
2. Ask questions about resume 
3. Find matching jobs
4. Track application progress
5. Practice interviews

The homepage should feel like a command center where the user chooses the next step in their job journey.

## The 5 Service Boxes

### 1) ATS Scorer
Purpose:
- Compares resume against a target job description and gives fit score.

What this card should communicate:
- "Check how well your resume matches a role"
- Visual hint: score/gauge icon style

Card content:
- Title: ATS Scorer
- Short text: "Get match score, missing keywords, and improvement suggestions."
- CTA: "Calculate Score"

### 2) Chat with Resume
Purpose:
- Lets user ask AI questions about their own resume.

What this card should communicate:
- "Talk to your resume"
- Visual hint: chat bubble icon

Card content:
- Title: Chat with Resume
- Short text: "Ask questions about skills, experience, strengths, and role fit."
- CTA: "Start Chat"

### 3) Job Search
Purpose:
- Finds remote jobs and can rank them by resume match.

What this card should communicate:
- "Find relevant jobs quickly"
- Visual hint: search or globe icon

Card content:
- Title: Job Search
- Short text: "Search remote roles and rank opportunities by your profile match."
- CTA: "Find Jobs"

### 4) Application Tracker
Purpose:
- Tracks all applications and statuses in one place.

What this card should communicate:
- "Manage your pipeline"
- Visual hint: checklist or kanban icon

Card content:
- Title: Application Tracker
- Short text: "Add, edit, and monitor applications with analytics."
- CTA: "Open Tracker"

### 5) Mock Interview
Purpose:
- Voice-based AI interview practice with final report.

What this card should communicate:
- "Practice before real interviews"
- Visual hint: microphone icon

Card content:
- Title: Mock Interview
- Short text: "Practice Q&A with AI and get performance feedback."
- CTA: "Start Interview"


## One-Line Product Description (for Design Header)
"An AI Job Assistant that turns an uploaded resume into actionable career steps: score, chat, search, track, and practice interviews."
